    DirectML 中的资源绑定1

    将资源绑定到 pipeline 上，DirectML 资源包含下面几种

    • Input
    • Output
    • Temporary
    • Persistent

    本文包含绑定的概念及实现细节

    重要概念

    执行 dispatchable（operator initializer/ compiled operator），需依照以下步骤进行。

    • IDMLDispatchable::GetBindingProperties 获取当前 dispatchable 所需的 descriptors 数量，temporary/persistent 资源

    • 创建足够容纳 descriptors 的 Direct3D 12 descriptor heap，绑定到 pipeline

    • IDMLDevice::CreateBindingTable 创建 DirectML 的 binding table，表示 pipeline 上的资源。使用 DML_BINDING_TABLE_DESC 结构来描述 binding table。

    • 创建 Direct3D 12 buffer 格式的 temporal/persistent 资源， 用 DML_BUFFER_BINDING 和 DML_BINDING_DESC 描述，添加到 binding table

    • 当 dispatchable 为 compiled operator 时，创建 Direct3D 12 buffer 资源，添加到 binding table

    • 将 binding table 作为参数传入 IDMLCommandRecorder::RecordDispatch

    获取 Binding Properties

    Note

    相同的 operator 可能会有不同的 binding properties

    DML_BINDING_PROPERTIES 结构定义了 dispatchable 绑定资源相关数据，下面是结构的完整定义

    struct DML_BINDING_PROPERTIES
{
    UINT RequiredDescriptorCount;
    UINT64 TemporaryResourceSize;
    UINT64 PersistentResourceSize;
};

    使用 IDMLDispatchable::GetBindingProperties 获取数据

    winrt::com_ptr<::IDMLCompiledOperator> dmlCompiledOperator;
// Code to create and compile a DirectML operator goes here.

DML_BINDING_PROPERTIES executeDmlBindingProperties{
    dmlCompiledOperator->GetBindingProperties()
};

winrt::com_ptr<::IDMLOperatorInitializer> dmlOperatorInitializer;
// Code to create a DirectML operator initializer goes here.

DML_BINDING_PROPERTIES initializeDmlBindingProperties{
    dmlOperatorInitializer->GetBindingProperties()
};

UINT descriptorCount = ...
    • RequiredDescriptorCount：Descriptor Heap size
    • TemporaryResourceSize
    • PersistentResourceSize

    创建 Descriptor Heap

    DirectML 在 Heap 上创建、管理 Descriptors。

    1. D3D12_DESCRIPTOR_HEAP_DESC 结构描述 Heap
    2. ID3D12Device::CreateDescriptorHeap 创建 Heap。
    3. ID3D12GraphicsCommandList::SetDescriptorHeaps 将 Heap 绑定到 pipeline。
    typedef struct D3D12_DESCRIPTOR_HEAP_DESC {
  D3D12_DESCRIPTOR_HEAP_TYPE  Type;
  UINT                        NumDescriptors;
  D3D12_DESCRIPTOR_HEAP_FLAGS Flags;
  UINT                        NodeMask;
} D3D12_DESCRIPTOR_HEAP_DESC;

    winrt::com_ptr<::ID3D12DescriptorHeap> d3D12DescriptorHeap;

D3D12_DESCRIPTOR_HEAP_DESC descriptorHeapDescription{};
descriptorHeapDescription.Type = D3D12_DESCRIPTOR_HEAP_TYPE_CBV_SRV_UAV;
descriptorHeapDescription.NumDescriptors = descriptorCount;
descriptorHeapDescription.Flags = D3D12_DESCRIPTOR_HEAP_FLAG_SHADER_VISIBLE;

winrt::check_hresult(
    d3D12Device->CreateDescriptorHeap(
        &descriptorHeapDescription,
        _uuidof(d3D12DescriptorHeap),
        d3D12DescriptorHeap.put_void()
    )
);

std::array<ID3D12DescriptorHeap*, 1> d3D12DescriptorHeaps{ d3D12DescriptorHeap.get() };
d3D12GraphicsCommandList->SetDescriptorHeaps(
    static_cast<UINT>(d3D12DescriptorHeaps.size()),
    d3D12DescriptorHeaps.data()
);

    创建 Binding Table

    Binding Table 用于表示 dispatchable 的资源。

    • Input
    • Output
    • Temporary
    • Persistent

    DML_BINDING_TABLE_DESC 用于描述 binding table

    struct DML_BINDING_TABLE_DESC {
  IDMLDispatchable            *Dispatchable;
  D3D12_CPU_DESCRIPTOR_HANDLE CPUDescriptorHandle;
  D3D12_GPU_DESCRIPTOR_HANDLE GPUDescriptorHandle;
  UINT                        SizeInDescriptors;
};

    IDMLDevice::CreateBindingTable 方法创建 DirectML binding table。之后我们将创建资源，并将这些资源添加到 binding table。

    DML_BINDING_TABLE_DESC dmlBindingTableDesc{};
dmlBindingTableDesc.Dispatchable = dmlOperatorInitializer.get();
dmlBindingTableDesc.CPUDescriptorHandle = d3D12DescriptorHeap->GetCPUDescriptorHandleForHeapStart();
dmlBindingTableDesc.GPUDescriptorHandle = d3D12DescriptorHeap->GetGPUDescriptorHandleForHeapStart();
dmlBindingTableDesc.SizeInDescriptors = descriptorCount;

winrt::com_ptr<::IDMLBindingTable> dmlBindingTable;
winrt::check_hresult(
    dmlDevice->CreateBindingTable(
        &dmlBindingTableDesc,
        __uuidof(dmlBindingTable),
        dmlBindingTable.put_void()
    )
);
    • DirectML 将 Descriptor 写入 Heap 的顺序未定义，注意不要覆写 binding table 中的 descriptor
    • 代码中注意在执行binding table 前，保证 CPU descriptors 拷贝到 GPU 中
    • Handle 支持 D3D12_DESCRIPTOR_HEAP_TYPE_CBV_SRV_UAV 类型的 descriptor heap
    • heap 必须是 shader-visible 的

    重置 binding table 移除所有资源

    1. 修改 binding table 描述
    2. IDMLBindingTable::Reset
    dmlBindingTableDesc.Dispatchable = pIDMLCompiledOperator.get();

winrt::check_hresult(
    pIDMLBindingTable->Reset(
        &dmlBindingTableDesc
    )
);

    绑定 Temporary/Persistent

    使用从 dispatchable 中获取的 binding 参数填充 DML_BINDING_PROPERTIES 结构。创建 Direct3D 12 buffer 资源，添加到 binding table 中。

    D3D12_HEAP_PROPERTIES defaultHeapProperties{ CD3DX12_HEAP_PROPERTIES(D3D12_HEAP_TYPE_DEFAULT) };
winrt::com_ptr<::ID3D12Resource> temporaryBuffer;

D3D12_RESOURCE_DESC temporaryBufferDesc{ CD3DX12_RESOURCE_DESC::Buffer(temporaryResourceSize) };
winrt::check_hresult(
    d3D12Device->CreateCommittedResource(
        &defaultHeapProperties,
        D3D12_HEAP_FLAG_NONE,
        &temporaryBufferDesc,
        D3D12_RESOURCE_STATE_COMMON,
        nullptr,
        __uuidof(temporaryBuffer),
        temporaryBuffer.put_void()
    )
);

DML_BUFFER_BINDING bufferBinding{ temporaryBuffer.get(), 0, temporaryResourceSize };
DML_BINDING_DESC bindingDesc{ DML_BINDING_TYPE_BUFFER, &bufferBinding };
dmlBindingTable->BindTemporaryResource(&bindingDesc);

    temporary 资源是 operator 执行过程中内部使用的存储资源（GPU Memory），程序不需要关心 temporary 资源的内容，也不需要在 IDMLCommandRecorder::RecordDispatch 执行后保留它。程序可以选择释放资源，或者在不同的 dispatch 上重用这块资源。

    对于 persistent 资源，创建流程与上述一致。但需使用 IDMLBindingTable::BindOutputs 将资源添加到 operator initializer 的 binding table 中，初始化 persistent 资源是 initializer 的职责，然后使用 IDMLBindingTable::BindPersistentResource 绑定到 compiled operator。persistent 资源的生命周期和 operator 一样长。

    persistent 资源通常用于存储查找表或 operator initialization 计算出的数据，这些数据被 operator 使用。

    绑定 Tensors

    compiled operator 需为 binding table 绑定 input 和 output 资源。

    用 DML_BUFFER_BINDING 和 DML_BINDING_DESC 结构来描述 input output 资源，使用 IDMLBindingTable::BindInputs 和 IDMLBindingTable::BindOutputs 添加到 binding table 里。调用 IDMLBindingTable::Bind* 方法时, DirectML 会将 descriptors 写入 CPU descriptors 中。

    DML_BUFFER_BINDING inputBufferBinding{ inputBuffer.get(), 0, tensorBufferSize };
DML_BINDING_DESC inputBindingDesc{ DML_BINDING_TYPE_BUFFER, &inputBufferBinding };
dmlBindingTable->BindInputs(1, &inputBindingDesc);

DML_BUFFER_BINDING outputBufferBinding{ outputBuffer.get(), 0, tensorBufferSize };
DML_BINDING_DESC outputBindingDesc{ DML_BINDING_TYPE_BUFFER, &outputBufferBinding };
dmlBindingTable->BindOutputs(1, &outputBindingDesc);

    使用 DML_TENSOR_FLAG_OWNED_BY_DML 标识，让 DirectML 管理 tensors。DirectML 将数据拷贝、存储到 persistent 资源，DirectML 将数据转化为更高效执行的格式。对于 operator 生命周期不变的数据（例如权重），可以提升性能。拥有该标识的的 tensor 只能在 initializer 阶段使用。

    对于非 DML_TENSOR_FLAG_OWNED_BY_DML 标识的 tensor 在执行 operator 时绑定。

    执行 Dispatchable

    使用 IDMLCommandRecorder::RecordDispatch 传递 binding table 参数。 程序需保证 CPU Descriptors 复制到了 GPU 中。

    winrt::com_ptr<::ID3D12GraphicsCommandList> d3D12GraphicsCommandList;
// Code to create a Direct3D 12 command list goes here.

winrt::com_ptr<::IDMLCommandRecorder> dmlCommandRecorder;
// Code to create a DirectML command recorder goes here.

dmlCommandRecorder->RecordDispatch(
    d3D12GraphicsCommandList.get(),
    dmlOperatorInitializer.get(),
    dmlBindingTable.get()
);

    关闭 command list，提交执行。

    GPU 执行 RecordDispatch 前，程序需要将所有绑定的资源转换到 D3D12_RESOURCE_STATE_UNORDERED_ACCESS 状态，或隐式转换到 D3D12_RESOURCE_STATE_UNORDERED_ACCESS 的状态，例如 D3D12_RESOURCE_STATE_COMMON。

    上述例外是 upload heaps 绑定到初始化 operator DML_TENSOR_FLAG_OWNED_BY_DML。这种情况下，upload heaps 应处在 D3D12_RESOURCE_STATE_GENERIC_READ 状态。

    对于 RecordDispatch，需要使用 unordered access view (UAV) barriers 来保证 dispatches 间正确数据依赖。示例代码中没有 UAV barriers，当 dispatch 间有数据依赖（后面 dispatch 的 input 用到了前面 dispatch 的 output），实际程序必须确保正确的 UAV barriers。

    Descriptors、Binding Table 的生命周期和同步

    • 当 Descriptor 被使用时（例如，被前一帧使用），binding table 无法覆写，程序需要等待 dispatchable GPU 上结束执行。
    • Binding table 不持有 descriptor heap 的强引用，因而程序应等待使用了这个 binding table 的 dispatchable GPU 执行结束后再释放 descriptor heap。
    • Binding table 不保证线程安全，不要在未同步情况下多线程使用 binding table。
    • 对于 ML、CG 混合工作流，程序应保证 binding table 指向的 descriptor heap 没有正被 GPU 使用。
