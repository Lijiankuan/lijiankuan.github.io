    Persistent Temporary 资源说明

    Persistent 资源与 compiled operator 生命周期一致。如果 operator 请求 persistent 资源，那么程序必须在 initializer operator 上提供，并且在之后的执行过程中这块资源不能被修改。2

    对于 initializer operator，output 上的绑定代表了 operator 的 persistent 资源。3

    DML_TENSOR_FLAG_OWNED_BY_DML 标识用于权重。

    If the dispatchable reports a non-zero size for its more long-lived persistent resource, though, then the procedure is a little different. You should create a buffer and describe a binding following the same pattern as shown above. But add it to your operator initializer's binding table with a call to IDMLBindingTable::BindOutputs, because it's the operator initializer's job to initialize the persistent resource. Then add it to your compiled operator's binding table with a call to IDMLBindingTable::BindPersistentResource.4

    For operator initializers, input bindings are expected to be of type DML_BINDING_TYPE_BUFFER_ARRAY with one input binding per operator to initialize, supplied in the order that you specified the operators during creation or reset of the initializer. Each buffer array should have a size equal to the number of inputs of its corresponding operator to initialize. Input tensors that had the DML_TENSOR_FLAG_OWNED_BY_DML flag set should be bound during initialize—otherwise, nothing should be bound for that tensor. If there is nothing to be bound as input for initialization of an operator (that is, there are no tensors with the DML_TENSOR_FLAG_OWNED_BY_DML flag set) then you may supply nullptr or an empty DML_BUFFER_ARRAY_BINDING to indicate 'no binding'.5

    Persistent 资源使用方法：

    1. 创建 persistent 资源
    2. 初始化时将 persistent 资源绑定到 initializer operator 的 outputs（执行初始化前需要将 weight tensors 绑定到 inputs）
    3. 执行时绑定到 compiled operator 的 persistent

  const DML_TENSOR_DESC     *InputTensor;
  const DML_TENSOR_DESC     *FilterTensor;
  const DML_TENSOR_DESC     *BiasTensor;
  const DML_TENSOR_DESC     *OutputTensor;
  DML_CONVOLUTION_MODE      Mode;
  DML_CONVOLUTION_DIRECTION Direction;
  UINT                      DimensionCount;
  const UINT                *Strides;
  const UINT                *Dilations;
  const UINT                *StartPadding;
  const UINT                *EndPadding;
  const UINT                *OutputPadding;
  UINT                      GroupCount;
  const DML_OPERATOR_DESC   *FusedActivation;
};

    Conv Operator 工作流

    初始化

    1. 创建 D3D12 device、queue、list、heaps，DML device

    创建 Operator

    1. 创建 InputTensor，FilterTensor，BiasTensor（可选），OutputTensor Tensor 描述符
    2. 用 1. 中描述符，创建 DML_CONVOLUTION_OPERATOR_DESC
    3. 创建 Operator

    初始化 Operator

    1. 创建 Temporary、Persistent 资源
    2. 绑定 Temporary 资源（Persistent 仅需要执行 Operator 时绑定）
    3. 为 binding table 绑定 Outputs（此处为 Persistent 资源
    4. 为 binding table 绑定 Inputs
    5. 初始化 Operator

    执行 Operator

    1. 绑定 Temporary、Persistent 资源
    2. 为 binding table 绑定 Inputs
    3. 为 binding table 绑定 InputsOutputs
    4. 执行 Operator

    结束

    1. 回读结果
