    《深入探索 C++ 对象模型》

    C++ 对象模型概念

    • 语言中直接支持面向对象程序设计的部分
    • 对于各种支持的底层实现机制

    关于对象

    C 语言

    • 不支持“数据”与“针对数据操作”的关联
    • 程序性的（procedural）

    C++

    • 抽象数据类型（ADT）
    • 参数化

    加上封装后的布局成本

    C++ 在布局及存取时间上的主要开销是由 virtual 引起的

    • virtual function 机制，用于支持执行时绑定
    • virtual base class，多次出现在继承体系中的 base class 有一个单一而被共享的实例

    此外也包括

    • derived class 和非第一 base class 之间的转换

    1.1 C++ 对象模型

    C++ 中的类包含以下内容：

    class data members

    • static
    • nonstatic

    class menber functions

    • static
    • nonstatic
    • virtual

    简单对象模型

    每个 slot 指向一个 members（data/function）。

    表格驱动对象模型

    • data member table
    • member function table

    C++ 对象模型

    class object 之外包含以下内容

    • static data
    • static function
    • non-static function

    Virtual functions 由以下方式支持

    • class 产生一堆指向 virtual functions 的指针，置入表格中，称为 virtual table（vtbl）
    • class object 中安插一个指向 virtual table 的指针，称为 vptr
    • vptr 设定和重置由每一个 class 的 constructor、destructor、copy assign 自动完成
    • 用以支持 RTTI 的，与 class 关联的 type_info object 也由 virtual table 指出，通常放在 vtbl 第一个 slot

    加上继承

    • 简单模型，object 含 bptr，指向基类 object

      • 间接性带来开销
      • class object 对继承都有一致的表现方式，class object 大小与 base classes 无关
      • 无需改变 class object，修改 base class table

    • C++ 最初继承模型不采用任何间接性，base class subobject 的 data members 直接放置在 derived class 中

      • base class members 任何改变，都需要重新编译

    • C++ 2.0 Virtual base class，加入了间接性，通过导入 virtual base class table，或扩充 virtual table 的方式

    对象模型如何影响程序

    程序在内部转换时，会修改已有代码，或加入新的代码。

    1.2 关键词所带来的差异

    与 C 兼容导致 C++ 编译解决方式复杂

    关键词的困扰

    struct 仅仅为了兼容 C，与 class 唯一的不同在于 struct 成员默认访问为 public，二者在编译层级共享一套代码。

    策略性正确的 struct

    • C++ class 中处于不同 access section 的数据排列顺序未定义
    • C++ class 中 base class 与 derived class 布局顺序未定义
    • virtual function 也会影响布局

    若需要复杂的 C++ class 某部分数据

    • 使用组合

      • conversion 运算符重载
    struct C_point{};

class Point{
public:
    operator C_point(){ return _c_point; }
private:
    C_point _c_point;
}

    1.3 对象的差异

    C++ 程序设计模式直接支持三种 programming paradigms

    • 程序模型

    • 抽象数据类型模型

      • 数据和其操作一起提供

    • 面向对象模型

      • 多态

    不同范式的混合导致行为不稳定

    • ADT 处理已知类型的实例，编译时期确定
    • OO 程序的多态，需通过 pointer/reference 间接处理
    • C++ 中多态仅存在于 public class 体系中

    C++ 通过以下方式支持多态

    1. 隐式转化
    2. virtual function
    3. dynamic cast 及 typeid

    多态通过共同接口影响类型封装，接口定义在 base class 中

    • 共享接口由 virtual function 机制完成

    class object 内存需求

    • non static data
    • members 之间的 alignment
    • virtual 产生的 overhead

    指针的类型

    转换是编译器指令，只影响“被指出内存的大小和其内容的解释方式”

    加上多态之后

    编译时期，指针类型决定以下两点

    • 可用接口
    • 接口的 access level

    执行时期，指针所指的 object 类型将决定所调用的实例（virtual 机制）

    面向对象的设计风格

    base class object 对象直接被初始化为 derived class object 时

    • 引起切割
    • 父类实例与子类无关
    • 多态失效

    OO 程序设计

    • 多态是一种强大的设计机制，允许继承抽象的 public 接口，封装相关的类型
    • 付出的是间接性，包括内存的获得/类型的决断
    • C++ 通过 pointers/references 支持多态

    ADT 程序风格

    • 包括数据和算法
    • 不支持类型的扩充

    OO & ADT 比较

    • ADT 速度快，空间紧凑（速度快由于函数调用编译期解析，空间紧凑由于不需要负担 virtual 机制带来的额外开销
    • OO 设计扩展性好
